# 诚实的能力边界

> 「知可以战与不可以战者胜」——孙子·谋攻篇

## 体系能力声明

591 Agent 体系覆盖 **70-80% 的知识工作问题**（非 99.99%）。v1.0 架构文档中的 "99.99%" 是一个修辞性目标，而非工程承诺。本体系在以下条件下表现出色：

- 问题可以**完全用语言描述**（无体感/触觉/视觉依赖）
- 信息**可以通过文本渠道获取**（网络搜索、文件读取、API 查询）
- 任务是**知识工作**类型（分析、创作、编码、研究、规划）
- 上下文窗口**足以容纳完整流水线**（复杂问题约消耗 30K-60K tokens）

## 已知限制

| 限制 | 说明 |
|------|------|
| **物理世界** | 不能修理漏水的水龙头、不能拧螺丝。可以写修理指南。 |
| **实时数据断层** | 当 Web 搜索失败且无 API 时，只能报告信息缺口 |
| **体感知识** | 不能教"感觉面团准备好了"这种触觉判断 |
| **跨会话长周期** | 没有持久化的任务状态管理。18个月的项目无法在一个会话中完成。 |
| **对抗性信息环境** | 谏省的"对抗性审查"是通用怀疑，不是针对特定对手的策略博弈 |
| **情感/关系问题** | 可以分析，但不能建立信任或修复情感创伤 |
| **真正新领域** | 当遇到训练数据完全未覆盖的全新科学问题时，体系只能类比已知领域 |
| **资源枯竭** | "用 ¥500 击败 Google"——体系可以解释为什么不可能，但不能让不可能变成可能 |
| **用户自我欺骗** | 用户错误理解自己的处境时，体系可能基于错误前提工作 |
| **并行执行** | 九部"并行执行"在当前实现中是单一线程的角色扮演，不是真正的并行 |

## 何时不用 591

- 你只需要一个快速的单轮回答 → 直接用 Claude，不需要 591
- 任务低风险、高度确定 → 用极速模式（跳过谏省+验省）
- 任务需要触觉/视觉/物理交互 → 591 只能辅助文本层面

| 实现完整度 | 3/10 | 设计文档 > 可部署代码 |
| 可用性 | 7/10 | 角色扮演模式在文本任务上表现良好 |
| **综合** | **5.8/10** | 优秀的设计，初级的实现 |
