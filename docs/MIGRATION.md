# 版本迁移指南

## 从 v2.1 → v2.2（当前）

v2.2 是向后兼容的增强版本，主要新增了：
- 版本元数据：所有 Agent 文件头部新增 `> v2.2 | 2026-05-31 | 591体系[层级]`
- 察省：独立「核心职责」章节
- 验省：容差范围概念
- `.claude/commands/`：5个斜杠命令文件

**无需迁移**：直接使用新版本即可。

## 从 v2.0 → v2.2

v2.1/v2.2 对 v2.0 做了大量增强，但所有 Agent 的 System Prompt 核心逻辑未变。主要变化：

1. **九部全部新增了「技能依赖」和「协作协议」章节** — v2.0 的九部文件如果被单独引用过，请更新引用路径。
2. **CLAUDE.md 文件名不再需要从 v2.0 迁移** — 路径兼容。
3. **五省文件新增了版本元数据行** — 不影响任何现有引用。

## 从 CLAUDE.md (v2.0) → CLAUDE-opt.md (v3.0 预览)

CLAUDE-opt.md 是实验性的 v3.0 优化版本，引入了：

- 二层流水线（轻量/标准/深度）替代四模式（标准/极速/严审/诊断）
- 两个新省：衡省（资源杠杆）、时省（时机策略）— 五省变七省
- 上下文预算管理系统
- 内嵌 Agent 提示词（替代外部文件引用）

**当前不建议迁移到 v3.0**。它代表 591 的未来方向，但尚未经过实战验证。v2.2 是生产推荐的版本。v3.0 中经过验证的概念会在未来版本中正式合并。

## 从三省六部（v1.0 外贸特定）→ 五省九部一院

这是架构级别的重构，不是版本升级。核心变化：

| 旧 | 新 | 变化原因 |
|----|----|---------|
| 中书省（策略+感知） | 察省+策省 | 感知和生成必须分离 |
| 门下省（审方案） | 谏省 | 审方案 → 审方案+需独立验证结果 |
| 尚书省（执行分发） | 协省 | 增强为 TL 角色 |
| 无 | 验省 | 旧架构缺失：无独立验证 |
| 无 | 进院 | 旧架构缺失：无自进化 |
| 六部（外贸环节） | 九部（问题域） | 领域特定 → 领域无关 |

旧三省六部的 Agent 定义已归档。新项目应使用五省九部一院。
